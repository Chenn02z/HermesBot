Root Causes & Fixes

🔴 Root Cause

1. Massive startup reads
AGENTS.md required reading 4 docs (~40KB) before any work. Every invocation re-reads them.

2. Main agent = bottleneck + highest cost model
Main agent (gpt-5.5) did 88% of events: reading source, editing files, running tests — all at premium rates.

3. No batching policy
Reviewer findings triggered one-at-a-time fix-verify cycles. 10 verification rounds for 3 issues.

4. Double verification
Main agent ran full pytest/ruff, then test-runner ran them again. Identical results.

5. SKILL.md verbosity
Skills restated workflow rules already in WORKFLOWS.md. tracer had full JSON schema.

🟢 Fix

1. Tiered startup
AGENTS.md says "read README once; SKILL.md declares prerequisites." Per-invocation reads: spec + SKILL.md + handoff interface (~5KB).

2. gpt-5.4 main + strict delegation
Main agent downgraded to gpt-5.4. Never reads source, never edits, never verifies. Only delegates, reconciles, reports.

3. Batched implementer passes
Reviewer findings are collected, deduplicated, sent as one implementer batch. One fix → one verify → done.

4. Single verification gate
test-runner is the sole verification path. Main agent reports results, never re-runs.

5. SKILL.md pruning
Removed restated workflow rules. tracer stripped of JSON schema. Total skills: 15.9KB → 8.6KB.

Delegation Guardrails (enforced in AGENTS.md + hermes-dev-loop SKILL.md)

During a dev-loop:
❌ Never read source files — explorer exists for that.
❌ Never edit files — implementer exists for that.
❌ Never run verification commands — test-runner exists for that.
✅ Only delegate, reconcile, report, and interact with the user.
✅ Batch all reviewer findings into one implementer pass — no fix-verify ping-pong.
